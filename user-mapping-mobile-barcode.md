# 財政部電子發票平台｜手機條碼 / 載具 User Flow

## 文件目的
整理財政部電子發票整合服務平台中，與「手機條碼」與「載具」相關的消費者功能，並轉成可供產品規劃、UX 設計與 prototype 參考的 user flow 文件。

> 資料來源：財政部電子發票整合服務平台公開 menu / portal JSON / 前端 route 與文案整理。

---

## 一、功能範圍

### 核心功能
- 手機條碼申請
- 手機條碼發票查詢
- 歸戶設定
- 載具專區
- 領獎設定
- 忘記驗證碼（密碼）

### 延伸功能
- 自然人憑證條碼專區
- 捐贈碼查詢
- 電子發票營業人查詢
- 加值服務中心查詢
- 線上申請

---

## 二、主要使用者角色

### 1. 新手使用者
特徵：
- 還沒有手機條碼
- 不理解載具與雲端發票概念
- 第一次接觸電子發票平台

需求：
- 快速理解手機條碼是什麼
- 完成申請
- 知道申請後能做什麼

### 2. 已申請手機條碼的使用者
特徵：
- 已有手機條碼
- 想查自己的發票
- 想確認發票有沒有進來 / 是否中獎

需求：
- 查詢發票
- 確認中獎狀態
- 查看發票明細

### 3. 進階載具使用者
特徵：
- 擁有多個會員卡或商家載具
- 想把不同通路發票集中管理

需求：
- 歸戶設定
- 載具管理
- 減少漏發票情況

### 4. 中獎 / 領獎導向使用者
特徵：
- 關心中獎後的收款流程
- 想先完成設定，避免中獎時再處理

需求：
- 設定領獎方式
- 確保領獎閉環完整

### 5. 登入受阻使用者
特徵：
- 忘記驗證碼
- 無法登入查詢或管理

需求：
- 快速找回驗證碼
- 恢復平台使用能力

---

## 三、核心 User Flow

### Flow 1｜新使用者申請手機條碼
**目標**：從 0 開始取得手機條碼，進入雲端發票系統

流程：
1. 進入電子發票整合服務平台
2. 點選「手機條碼申請」
3. 輸入手機號碼 / 基本資料
4. 完成身分驗證
5. 設定驗證碼（密碼）
6. 取得手機條碼
7. 開始用手機條碼存發票

完成結果：
- 擁有個人手機條碼
- 可登入查詢發票
- 可進一步做歸戶與領獎設定

可能卡點：
- 不理解手機條碼用途
- 驗證流程繁瑣
- 不知道申請完下一步是什麼

---

### Flow 2｜已申請手機條碼，查詢發票
**目標**：查看自己的雲端發票與發票明細

流程：
1. 進入平台
2. 點選「手機條碼發票查詢」
3. 輸入手機條碼 / 驗證碼登入
4. 進入查詢頁
5. 查看發票列表
6. 點開某張發票查看明細 / 狀態 / 中獎資訊

完成結果：
- 確認發票是否存在
- 查詢發票詳細內容
- 確認是否中獎或待開獎

可能卡點：
- 登入失敗
- 某張發票找不到
- 看不懂載具 / 發票來源差異

---

### Flow 3｜找不到發票，回頭確認是不是沒歸戶
**目標**：確認為什麼發票沒有進手機條碼帳戶

流程：
1. 使用者查詢發票
2. 發現某張發票沒出現
3. 進入「歸戶設定」或「載具專區」
4. 檢查是否已有綁定其他會員載具
5. 若尚未綁定，開始新增歸戶
6. 完成後，後續發票會集中到手機條碼

完成結果：
- 釐清發票缺漏原因
- 建立集中收納機制
- 降低之後再次漏發票的機率

可能卡點：
- 不知道「找不到發票」其實常常是「沒歸戶」
- 不知道哪些會員載具可綁定
- 歸戶名詞不直觀

---

### Flow 4｜新增載具歸戶
**目標**：把商家會員載具 / 其他載具綁到手機條碼

流程：
1. 進入「歸戶設定」
2. 選擇欲歸戶的載具 / 商家會員載具
3. 輸入必要的帳號驗證資訊
4. 完成綁定
5. 系統將該載具發票歸到手機條碼帳戶

完成結果：
- 發票可集中管理
- 不同通路的發票統一查詢
- 提高中獎發票不漏接的機率

可能卡點：
- 不清楚「歸戶」與「載具新增」差異
- 商家驗證流程麻煩
- 綁定成功後缺乏明確回饋

---

### Flow 5｜查看與管理載具
**目標**：查看目前有哪些載具，並管理設定

流程：
1. 進入「載具專區」
2. 查看已綁定載具
3. 檢查各載具狀態
4. 必要時調整 / 維護
5. 返回查詢發票或其他設定

完成結果：
- 使用者知道自己有哪些載具
- 可管理發票收納來源
- 強化對整體系統的掌控感

可能卡點：
- 載具種類命名太技術
- 使用者分不清手機條碼與其他載具層級

---

### Flow 6｜設定領獎方式
**目標**：中獎後可順利領獎 / 匯款

流程：
1. 進入「領獎設定」
2. 輸入銀行 / 匯款相關資料
3. 儲存設定
4. 之後中獎時依設定處理

完成結果：
- 建立領獎閉環
- 降低中獎後再補設定的麻煩
- 提高領獎成功率

可能卡點：
- 對帳戶安全有疑慮
- 不知道為何要先設定
- 不清楚設定後的實際效果

---

### Flow 7｜忘記驗證碼，重新取得登入能力
**目標**：恢復登入，重新使用所有功能

流程：
1. 使用者要查發票 / 設定載具
2. 發現忘記驗證碼
3. 點選「忘記驗證碼(密碼)」
4. 進行身分驗證
5. 重設驗證碼
6. 回到登入
7. 繼續查詢 / 歸戶 / 領獎設定

完成結果：
- 帳號可重新使用
- 使用者不會因密碼遺失而流失

可能卡點：
- 驗證程序太長
- 不知道「驗證碼」其實就是密碼

---

### Flow 8｜查詢外部資訊輔助使用
**目標**：補充使用過程需要的外部資訊

流程：
1. 使用者不確定某商家 / 功能 / 捐贈方式
2. 進入以下功能：
   - 捐贈碼查詢
   - 電子發票營業人查詢
   - 加值服務中心查詢
3. 查詢相關資訊
4. 再回到主要流程（查發票 / 歸戶 / 使用載具）

完成結果：
- 降低資訊不透明
- 幫助使用者做判斷

## 四、主流程總結

### 主幹流程
申請手機條碼 → 登入查發票 → 發現缺漏 → 做歸戶設定 → 載具集中管理 → 設定領獎 → 長期使用

### 產品拆模組建議
1. Onboarding
   - 手機條碼申請
2. Authentication
   - 登入
   - 忘記驗證碼
3. Invoice Management
   - 手機條碼發票查詢
4. Carrier Management
   - 歸戶設定
   - 載具專區
5. Prize Claim
   - 領獎設定

## 五、最值得做進 prototype 的三條 flow

### A. 新手主線
申請手機條碼 → 查第一張發票

### B. 關鍵問題主線

### C. 完整閉環主線
查到中獎發票 → 領獎設定
